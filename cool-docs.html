Cool Language
Documentation

A Python-inspired scripting language and interactive OS shell, implemented in Rust. Clean syntax, closures, classes, f-strings, and a fully-featured shell — all in one.

Get Started View on GitHub

Introduction

Cool is a tree-walk interpreted language with Python-like syntax built on a clean Rust runtime. It ships with Cool — a fully interactive OS shell written entirely in Cool itself.

Indentation Blocks
Python-style. No braces, no end keywords.
Classes & OOP
Inheritance, super(), operator overloading.
Closures
Functions capture their scope. nonlocal & global supported.
f-strings
Interpolation, expressions, method calls inline.
Standard Library
math, json, re, time, random, collections, path, subprocess, argparse, logging, term.
Cool Shell
Full interactive shell. Pipes, env vars, tab complete.
Rust Runtime
Fast, safe, no external dependencies.
MIT Licensed
Free to use, modify, and distribute.

Closures

cool
def make_counter(start=0):
    count = start
    def increment(by=1):
        nonlocal count
        count += by
        return count
    return increment

c = make_counter(10)
c()     # 11
c(5)    # 16

nonlocal / global

cool
count = 0

def increment():
    global count     # access module-level variable
    count += 1

def outer():
    x = 0
    def inner():
        nonlocal x   # access enclosing function's variable
        x += 1
    inner()
    return x         # 1

__enter__ / __exit__ with obj as x:
cool
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __mul__(self, scalar):
        return Vector(self.x * scalar, self.y * scalar)

    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

    def __len__(self):
        return 2

    def __str__(self):
        return f"({self.x}, {self.y})"

v1 = Vector(1, 2)
v2 = Vector(3, 4)
print(v1 + v2)     # (4, 6)
print(v1 * 3)      # (3, 6)
print(len(v1))     # 2

Context Managers (with)

Custom __enter__ / __exit__ context managers work in both the tree-walk interpreter and the bytecode VM, including cleanup on normal exit, control-flow exits, and raised exceptions. LLVM native builds now support custom context managers and with open(...) on normal exit, control-flow exits, caught exceptions, and unhandled native raises too.

cool
class Timer:
    def __enter__(self):
        self.start = time.monotonic()
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        elapsed = time.monotonic() - self.start
        print(f"Elapsed: {elapsed:.4f}s")
        return false   # don't suppress exceptions

with Timer():
    do_heavy_work()

Error Handling

try / except / else / finally

cool
try:
    result = int(user_input)
    value  = 100 / result
except ZeroDivisionError:
    print("cannot divide by zero")
except ValueError as e:
    print(f"not a valid number: {e}")
except Exception as e:
    print(f"unexpected error: {e}")
else:
    print(f"result = {value}")   # runs only if no exception
finally:
    print("always runs — cleanup here")

raise

cool
def divide(a, b):
    if b == 0:
        raise ValueError("divisor cannot be zero")
    if type(a) != int or type(b) != int:
        raise TypeError("arguments must be integers")
    return a / b

Exception Propagation

Uncaught exceptions propagate up the call stack automatically:

cool
def parse(data):
    return int(data["value"])   # may raise ValueError

def process(raw):
    return parse(raw)           # exception propagates up

try:
    process({"value": "bad"})
except ValueError as e:
    print(f"caught it: {e}")

ffi (Foreign Function Interface)

Call native C functions from shared libraries. This enables low-level systems programming and access to native libraries.

cool
import ffi

# Open a shared library
libm = ffi.open("libm")           # math library on Unix
libc = ffi.open("libc")          # C standard library

# Define a function: ffi.func(lib, name, return_type, [arg_types])
sqrt_fn = ffi.func(libm, "sqrt", "f64", ["f64"])
abs_fn  = ffi.func(libc, "abs",  "i32", ["i32"])

# Call the native function
sqrt_fn(2.0)    # 1.414213...
abs_fn(-42)     # 42

File I/O

Opening Files

Mode Description
"r" Read (default). File must exist.
"w" Write. Creates file or truncates existing.
"a" Append. Creates file if it doesn't exist.

with Statement (Recommended)

Use with for structured file cleanup. Both the tree-walk interpreter and the bytecode VM close files on normal exit and raised exceptions. LLVM native builds now support open() and file context managers on normal exit, control-flow exits, caught exceptions, and unhandled native raises too.

cool
# Read entire file
with open("data.txt", "r") as f:
    content = f.read()

# Read lines
with open("data.txt", "r") as f:
    lines = f.readlines()
    for line in lines:
        print(line.strip())

# Write
with open("output.txt", "w") as f:
    f.write("Hello, file!\n")
    f.write("Second line\n")

# Append
with open("log.txt", "a") as f:
    f.write(f"[{time.time()}] Event occurred\n")

How the Interpreter Works

Cool uses a classic pipeline architecture:

  1. Lexer (lexer.rs) — scans source text into tokens. Handles Python-style indentation by emitting synthetic INDENT and DEDENT tokens.
  2. Parser (parser.rs) — recursive descent parser that consumes the token stream and produces an Abstract Syntax Tree (AST).
  3. Interpreter (interpreter.rs) — walks the AST and evaluates each node. Manages scope, closures, classes, and all built-in operations.

Phase 8 adds a compiler backend alongside the interpreter:

  1. Compiler (compiler.rs) — translates the AST into a flat bytecode instruction sequence.
  2. VM (vm.rs) — stack-based virtual machine that executes bytecode. Defined opcodes live in opcode.rs.

Phase 8 — Compiler

The compiler backend compiles Cool to native binaries, enabling self-hosting and OS-level use.

  • ✅ Bytecode VM — compile AST to bytecode, run on a stack-based virtual machine (compiler.rs, vm.rs, opcode.rs)
  • ✅ LLVM backend — compile a large native subset of Cool → LLVM IR → native binary
  • ✅ FFI — available in the interpreter and LLVM native builds via import ffi
  • ✅ Inline assembly — asm("template") blocks
  • ✅ Pointer types and raw memory access (malloc, free, read_i64, write_i64)
  • ✅ cool build command for compiling projects
  • ✅ Classes in LLVM — class, inheritance, __init__, methods, attribute access, super()
  • ⚠️ LLVM supports custom with / context managers and with open(...) on normal exit, return/break/continue, caught exceptions, and unhandled native raises, plus native try/except/finally/raise, regular project/module imports including dotted package imports, built-in subprocess.run()/call()/check_output(), argparse.parse()/help(), logging.basic_config()/info()/warning()/error(), and import ffi, but still excludes closures/lambdas

Phase 9 — Self-Hosted Compiler

The compiler is now written entirely in Cool itself, enabling Cool to compile itself!

  • ✅ Lexer in Cool (coolc/compiler_vm.cool)
  • ✅ Recursive descent parser in Cool
  • ✅ Code generator (AST → bytecode) in Cool
  • ✅ Bytecode VM in Cool (to execute compiled programs)
  • ✅ Bootstrap: self-hosted compiler compiles itself
